## Intended Integration: Fireflies.ai

Real implementation would:
1. On interview scheduled → call Fireflies API to register meeting:
   POST https://api.fireflies.ai/graphql
   { addToCalendar: { meeting_link, meeting_name, attendees } }
2. Fireflies bot joins automatically at scheduled time
3. Post-meeting webhook fires to POST /api/interviews/transcript
4. Payload includes: transcript, summary, action_items, speaker_labels
5. We store and process via same Claude analysis pipeline

Current implementation:
- Simulates note capture using Claude-generated, candidate-specific notes
- Timed reveals mimic real-time capture
- Full transcript + summary generated by Claude on interview end
- All downstream processing (bias check, recommendation, actions) 
  is production-ready

Limitation: Bot cannot join a real meeting in this prototype.
With Fireflies API key and a live meeting URL, the integration
would be active in under 2 hours of additional work.
